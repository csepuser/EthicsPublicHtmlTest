Dennis,

I just got your response to the code of ethics. I agree with many of the concerns expressed. The original plan was to distinguish codes of ethics from codes of conduct and codes of practice.

The code you have seen and commented on was developed by a sub?group led by Ed Mechler, of my working group on professional competence. I was surprised when it was distributed to the steering committee before going through the kinds of revision and distribution we did with the ACM code of ethics. (But I guess, sending it to directly to the top is one way to get things moving.)

I am concerned about your suggestion to approve by November. There are some senstive issues that need to be worked out and some restructuring of the code that needs to be done before it is ready. For example, given the current ambiguous state of copyright legislation, that item has to be filled out in the code. Are we ultimately going to have three types of code- ethics, conduct, aand practice or follow the ACM model of a code and put standards of conduct in the guidelines. For a code to be effective, there needs to be sanctions such as removal from membership in a society. Which society is sponsoring this and will there be similar sanctions from each society. Before the ACM code was approved it was submitted to the mebership in the CACM and responses were solicitied. I am beloboring my point - one month from draft to approval seems rather fast.

I think we have a good start and would like to develop it further. I will be attending an international computer ethics conference in November and I would like to share it with some of our international membership.

BTW, I have found a new use for the House Building video. I am teaching telecommunications protocol verification at George Washington University ( I'm on sabbatical). I used the tape to illustrate the requirements of concurrent processing and the virtues of it when it is done correctly. And as a case of overuse, I also used in my software project management class. It graphically makes the points about scheduling, resources, planning, estimation and a project plan. The tape is a marvelous resource.

best wishes, don gotterbarn
