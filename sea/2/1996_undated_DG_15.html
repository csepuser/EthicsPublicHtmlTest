This is a great first cut, suggestions for SEEPP Email to Robert Melford: "This is a great first cut, suggestions for SEEPP" - undated, 1996 (Level 2) E-mail

Bob,

This is a great first cut, but I think there are some important things that are needed, Here is a list of points. The critical ones are 1, 2, 5, 8

1. Our -the seep- schedule has to be revised and made known to the work groups so that they have some type of target. The balloting process will consume a significant amount of time which I don't think we planned for. 4.3.2 has the task force conduct a ballot- we don't seem to have any provisions for consensus other than the ballot. Shouldn't consensus be achieved in the wg before something gets sent to ballot?

2. Since all of the work group's reports will be grouped together in a single seep product they should have a common style. WE NEED TO PROVIDE A STYLE( see also point 5). If we could break the outline out a little more, it would help. I dont think we are going for Standards. If we are not strictly sticking to the IEEE S tandards model, the distinction between upper case S and lower case s in section 3.1 and 3.2 is not relevant.

3. Section 4.1 says work groups deliver their products to the BOG, dont they deliver them to the task force first.

4. section 4.1.2 defines a Quorum, but it does not get used anywhere, e.g, affirmative votes etc are 70% of the balloting group.

5. Since this is going to be a task force report- we should define the scope(4.1.4) of the wg reports. The WHY will be the same for each wg, the only difference will be in their subject matter. This will give the groups the guidance necessary.

6. 4.1.4.2 The outline is not an item listed in STYLE

7. I have some concern about 4.3.6. some folks will probably object to their name being circulated with their negative votes. 8. I think the membership clauses are too restrictive. We would lose severl people from the task force is we followed those rules. Several did not make it to Boston or to Gatlinburg. We should soften the rules. When some see these rules they will drop out.

9. 4.2.3 does not match what we did. WG chairs were appointed by the Task force.

10. 4.3.5 c) last paragraph- chage second to last sentence to " when a 75% return of the balloting group is received, which ever is later." This allows for more the 75% affirmative votes before the balloting is closed.

Perhaps we could "borrow" some style and scope terminology from the code of ethics work group or one of the other groups like technology and society.
