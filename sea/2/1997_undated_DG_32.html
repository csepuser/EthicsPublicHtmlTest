SIGSOFT can participate in this effort Email from Gotterbarn to Lori Clarke and William Tracz: "SIGSOFT can participate in this effort" - Undated, 1997 (Level 2) E-mail

Dear Lori Clarke and William Tracz:

The IEEE-CS and the ACM have a joint steering committee to evaluate, plan, and coordinate actions related to establishing software engineering as a profession. It is composed of eminent software engineers and academics, such as Barry Boehm. The goal of this committee is "To establish the appropriate set(s) of standards for professional practice of Software Engineering upon which industrial decisions, professional certification, and educational curricula can be based." The steering committee decided to accomplish these tasks through the establishment of three series of task forces: Software Engineering body of knowledge and recommended practices; Software Engineering ethics and professional practices, and Software Engineering curriculum.

I am the chair of the Software Engineering ethics task force whose purpose is to document the ethical and professional responsibilities and obligations of software engineers. Like other professions, we have used a code of ethics to document these responsibilities and obligations.

The development of a code is an iterative process requiring multiple reviews in order to express a consensus of the profession. At this time we have a draft code which has gone through several internal reviews. It is now time to distribute the code to a broader contingency. SIGSOFT represents a significant contingent of software engineers and we would like to publish the draft of the code for review and comment by your membership. This type of review is necessary if the code is truly to represent the profession. It would be greatly appreciated if you could help us in the development of the code.

Please let me know if SIGSOFT can participate in this effort and what would be the best way to facilitate publication of the draft.

Thank you.

Sincerely,

don gotterbarn
