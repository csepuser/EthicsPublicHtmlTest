Proposal to SWECC - Software Engineering Professionalism and Ethics Project (SEPEP) Proposal to SWECC - Software Engineering Professionalism and Ethics Project (SEPEP) - April, 1999 (Level 2) Report

April1999 proposal to SWECC
Software Engineering Professionalism and Ethics Project (SEPEP)
April 1999 [Pending ACM and IEEE CS Approval]

Purpose of the Project
· Propose and undertake efforts to publicize the Software Engineering Code of Ethics.
· Foster broader acceptance of the Code across national and international borders.
· Act as laison to other national and international organizations to foster the Code.
· Update and maintain the Code as appropriate
· Respond to changing technologies and legislation to maintain the Code.
· Interact with the Software Engineering Education effort of SWECC incorporating knowledge of the Code and use of the Code in decision making in Software Engineering Education.

Structure and Governance
· The project would be on going.
· The project will be governed by an executive committee of three members appointed by SWECC.
· The chair must be a member of both the ACM and the Computer Society.
· The other two members will represent the ACM and the Computer Society respectively.
· The project will propose and undertake tasks related to its purposes.
· The fiscal year for SEPEP shall be July 1 to June 30.
· The chair will report annually on the work of the project to the SWECC, to the ACM and to the Computer Society.


Immediate Projects
· To support the educational goals of the SEPEC, develop a flyer about the Code to be included with all ACM and Computer Society membership renewals.
· Present a panel on the Code at the next SIGSOFT (ACM Special Interest Group on Software Engineering) and at the next TCSE (Computer Society Technical Committee on Software Engineering) meetings to demonstrate the effect of the Code in decision making and to develop a list of future modifications.
· Publish the Code in the CACM and Computer to forward the education purposes of the project and to establish archival copies.
· Work with the Education group to develop Code of Ethics educational materials.
· Establish and maintain a joint ACM/Computer Society web site on the Code.*
* The site would provide electronic access to the Code and contain discussions. It would provide access to a formatted copy of the Code which is designed for printing by individuals. It would also contain information about using the Code in decision making.


Initial Committee membership
· Donald Gotterbarn-Director of the Software Engineering Ethics Research Institute (ACM & IEEE-CS)
· Keith Miller-Software Engineering Professor(ACM & IEEE-CS)
· Simon Rogerson-Director of the Centre for Computing and Social Responsibility and Chair of Computer Ethics at DeMonfort University (ACM & BCS)


This group is the executive committee of the task force that successfully developed the Software Engineering Code of Ethics and Professional Practice. Their recent experience with the Code, backgrounds in software development and education, and experience with significant software development means they can immediately start on the work of the committee. Gotterbarn and Miller are members of both the Computer Society and the ACM.
